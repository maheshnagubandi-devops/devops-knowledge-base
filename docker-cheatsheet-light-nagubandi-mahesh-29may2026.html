🐳 What is Docker?   Docker allows developers to package applications along with their libraries & dependencies into a standardised unit called a container. Containers are platform-independent — they run the same on any machine. They allow applications to run in resource-isolated processes, eliminating infrastructure dependency issues ("works on my machine" problems). Unlike VMs, containers are lightweight and share the host OS kernel.

Dockerfile Instructions
📋 What is Dockerfile?
Plain text file with commands to build an image. Each command creates a layer. All layers together = one image.
FROMubuntu:22.04 (PYTHON:3.5/php:3.5/CENTOS:7/UBUNTU:latest)
RUNapt-get update
COPY./app /app
ADDfile.tar.gz /app# also downloads URL + auto-extracts
WORKDIR/app
ENVPORT=8080 ENV=prod
ARGBUILD_VER=1.0
EXPOSE8080
CMD["node","app.js"]
ENTRYPOINT["./run.sh"]
MAINTAINERMAHESH
if src is an URL,ADD will automatically download and extracts (if its zipped) to target path.
ARG Build-time only
ENV Build + runtime
CMD Can be overridden
ENTRYPOINT Fixed entry (use --entrypoint to override at runtime)
💡 CMD vs ENTRYPOINT: CMD = default execution (can be overridden). ENTRYPOINT = entry point (more stable).
🏗️ BEST PRACTICES:
✓ One service per container   ✓ Build context as small as possible
✓ Avoid unnecessary packages   ✓ Minimize number of layers

Key Concepts & Interview Points
Q Can a container restart by itself?
No bydefault, Yes — using restart policies: off on-failure always unless-stopped
Q docker info shows what?
Kernel version, total images, running containers, storage driver, runtime info
Q Difference: EXPOSE vs PUBLISH (-p)?
EXPOSE = container-to-container only. -p = publish to outside world. Publish = expose + publish.
Q ARG vs ENV?
ARG available build-time only. ENV available at both build & runtime.
Q docker attach vs docker exec?
attach connects to main process (exit closes container). exec creates new process (exit only closes session).
Q How does Docker achieve isolation?
Namespaces (PID, NET, UTS, IPC, USER) + cgroups (CPU, Memory, Disk I/O, Network bandwidth, Processes)
Q Volumes vs Bind Mounts?
Volumes = managed by Docker (/var/lib/docker/volumes). Bind mounts = managed by host, any path.

Docker vs VM vs Hypervisor vs Container Runtime
Docker
containers Virtualize the Software layers above the OS.
Uses a container runtime / execution engine on the host.
Shares the host OS kernel, no separate OS per container
Packages applications along with their dependencies.
Lightweight and fast startup(boot time).
More portable and resource-friendly.
Depends on the host operating system kernel.
Best foe micro services and CICD pipelines.
Container Runtime
A container runtime manages the lifecycle of containers.
Responsibilities
Create, start, stop, and delete containers.
Provide isolation using namespaces and cgroups.
Share the host OS kernel.
Examples:containerd,CRI-O
Virtual Machine (VM)
VM virtualize the entire machine including Hardware, Storage, Network, CPU, operating system
Uses a hypervisor to run a full guest operating system.
Each VM has:
Its own OS,Its own kernel,Virtualized hardware
Provides stronger isolation.
Higher resource and storage overhead.
Slower boot time compared to containers.
Hypervisor (VMM – Virtual Machine Monitor)
A hypervisor creates and manages virtual machines.
Responsibilities
Allocates CPU, memory, storage, and networking to VMs.
Provides virtual hardware to guest operating systems.
Enables multiple VMs to run on the same physical machine.
Types of Hypervisors
Type 1 Hypervisor (Bare-Metal)
Runs directly on physical hardware.
Does not require a host OS.
Can directly access hardware and networking.
Better performance and security.
Examples
VMware ESXi
Microsoft Hyper-V
Xen
Type 2 Hypervisor (Hosted)
Runs on top of an existing host operating system.
Uses the host OS for hardware access.
Easier for desktop/lab usage.
Examples
Oracle VirtualBox,VMware Workstation

Key Concepts & Interview Points
Q Can a container restart by itself?
No bydefault, Yes — using restart policies: off on-failure always unless-stopped
Q docker info shows what?
Kernel version, total images, running containers, storage driver, runtime info
Q Difference: EXPOSE vs PUBLISH (-p)?
EXPOSE = container-to-container only. -p = publish to outside world. Publish = expose + publish.
Q ARG vs ENV?
ARG available build-time only. ENV available at both build & runtime.
Q docker attach vs docker exec?
attach connects to main process (exit closes container). exec creates new process (exit only closes session).
Q How does Docker achieve isolation?
Namespaces (PID, NET, UTS, IPC, USER) + cgroups (CPU, Memory, Disk I/O, Network bandwidth, Processes)
Q Volumes vs Bind Mounts?
Volumes = managed by Docker (/var/lib/docker/volumes). Bind mounts = managed by host, any path.

Docker vs VM vs Hypervisor vs Container Runtime
Docker
containers Virtualize the Software layers above the OS.
Uses a container runtime / execution engine on the host.
Shares the host OS kernel, no separate OS per container
Packages applications along with their dependencies.
Lightweight and fast startup(boot time).
More portable and resource-friendly.
Depends on the host operating system kernel.
Best foe micro services and CICD pipelines.
Container Runtime
A container runtime manages the lifecycle of containers.
Responsibilities
Create, start, stop, and delete containers.
Provide isolation using namespaces and cgroups.
Share the host OS kernel.
Examples:containerd,CRI-O
Virtual Machine (VM)
VM virtualize the entire machine including Hardware, Storage, Network, CPU, operating system
Uses a hypervisor to run a full guest operating system.
Each VM has:
Its own OS,Its own kernel,Virtualized hardware
Provides stronger isolation.
Higher resource and storage overhead.
Slower boot time compared to containers.
Hypervisor (VMM – Virtual Machine Monitor)
A hypervisor creates and manages virtual machines.
Responsibilities
Allocates CPU, memory, storage, and networking to VMs.
Provides virtual hardware to guest operating systems.
Enables multiple VMs to run on the same physical machine.
Types of Hypervisors
Type 1 Hypervisor (Bare-Metal)
Runs directly on physical hardware.
Does not require a host OS.
Can directly access hardware and networking.
Better performance and security.
Examples
VMware ESXi
Microsoft Hyper-V
Xen
Type 2 Hypervisor (Hosted)
Runs on top of an existing host operating system.
Uses the host OS for hardware access.
Easier for desktop/lab usage.
Examples
Oracle VirtualBox,VMware Workstation
